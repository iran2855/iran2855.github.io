首先明确，kernel是在GPU上运行的特殊函数，每个thread是执行kernel的基本单位，block是thread的集合，grid是block的集合。而以上概念都是逻辑和抽象的，在硬件级别上没有直接对应的实体，但它们与硬件的实际执行密切相关，并由硬件特性和架构直接支持。

下面来介绍相关的硬件支持：

  • GPU core/Cuda core：GPU最基本的处理单元，GPU的cores负责执行threads。

  • SM(Streaming MultiProcessor)：一个SM由多个core组成，每个SM根据GPU架构不同有不同数量的core。SM还包括共享内存(shared memory)，寄存器文件(Register File)等，这就解释了为什么shared memory

直观感受如图：

注意：

  • 1、一个block一定全在一个SM上，但是一个SM上可以运行多个block

  • 2、定义block是程序员在code中做的，而GPU负责的是把blocks分配到SMs上

  • 3、thread是并行运行的，而为了保证程序运行的效率、简易性、规模性，cuda对于thread blocks运行的时间和位置几乎不做保险。

  • 4、一般而言，block之间没有交流。这会导致dead lock的发生（dead lock：如果一组进程中的每个进程都在等待一个事件，而这个事件只能由该组中的另一个进程触发，这种情况会导致死锁）
